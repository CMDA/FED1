Het begin

  • Sir Tim Berners-Lee is de uitvinder van het WWW en HTML.
  • Ontwikkelde op basis van SGML definitie voor HTML.
  • Plus een browser, WorldWideWeb, voor weergave.

Informatie delen

  • Ontwikkeld voor het gezamenlijk
    delen & organiseren van informatie.
  • Eenvoud, hyperlinks & juist tijd
    (toegang internet wijder verspreid +
    DNS).

Browsers

Nodig om HTML te kunnen renderen.

  • Mosaic (1993, img & forms)
  • Netscape (1994, Javascript)
  • Opera (1995)
  • Internet Explorer (1995, licenseerde Mosaic engine)
  • Safari, Firefox, Chrome...

In de tussentijd...

De eerste HTML specificatie in 1995 door het IETF uitgegeven onder de noemer HTML 2.0.

Hierin table, map & img elementen en form uploads.

Browser wars

1995: Netscape meest gebruikte browser, Internet Explorer met Windows meegevelerd en later gratis.

Features van elkaar overgenomen
Er worden HTML features buiten de specificatie om ingevoerd.

Eindigt in Internet Explorer als grote winnaar - nauwlijks andere browsers op het PC platform in gebruik.

Browser wars

    Gevolgen:
  • Innovatie in browsers verminderde - het web stond stil.
  • Web authors moeten nog steeds browser specifieke code schrijven (IE fixes).

Browser wars

Vanaf 2002 weer gedegen concurrentie voor IE.

  • Firefox, op basis van Gecko (Netscape) layout engine.
  • Chrome en Safari op basis van Webkit layout engine.

Ondanks verschillende browsers kan een rendering engine weer hetzelfde gaan doen als IE.

In de tussentijd...

De W3C - World Wide Web Consortium - neemt HTML specificatie taken over van IETF.

De W3C stelt specificaties op voor HTML 3, 3.2 en uiteindelijk 4.0, waarin veel onderdelen eerst door browser ontwikkelaars geïmplementeerd.

Maar ook: specifieke browser onderdelen komen niet in de specificaties.

Wanneer is de specificatie klaar?

Werkwijze W3C

  • Beslissingen worden genomen op
    basis van consensus.
  • Issues worden aangedragen,
    bediscussieerd en over gestemd.
  • Klinkt democratisch, maar politiek
    en interne conflicten vertragen
    het proces.

Werkwijze WHATWG

  • Ook hier worden issues aangedragen & bediscussieerd.
  • Echter is er één editor die beslist wat er in de HTML5 specificatie gaat.
  • Resulteert in een sneller proces.

W3C & WHATWG

  • 2009: W3C staakt ontwikkeling specificatie XHTML 2.
  • W3C gebruikt specificaties van WHATWG als basis voor HTML5 specificatie.
  • Specificatie mogelijk klaar in 2022.
